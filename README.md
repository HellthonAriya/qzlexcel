# qzlexcel — ربات تلگرام مدیریت پیگیری دانش‌آموزان

ربات تلگرامی برای نمایش، ویرایش و پیگیری وضعیت تماس با دانش‌آموزان از روی یک فایل اکسل چندشیتی، با پجینیشن، جستجو، دکمه‌های شیشه‌ای کپی سریع شماره تلفن، و خروجی اکسل آنی از تغییرات. شیت‌ها هرچه باشند و به هر نامی، ربات خودش آن‌ها را شناسایی می‌کند؛ نیازی به نام یا تعداد ثابت شیت نیست.

## امکانات

- **پجینیشن**: نمایش ردیف‌ها صفحه‌به‌صفحه (تعداد قابل تنظیم در هر صفحه) با دکمه‌های اول / قبلی / بعدی / آخر.
- **متن هر صفحه**: دقیقاً مشخص می‌کند از چه شماره ردیفی تا چه شماره ردیفی (بر اساس ستون «ردیف» خود فایل) در حال نمایش است.
- **دکمه‌های شیشه‌ای کپی شماره**: برای شماره خودِ دانش‌آموز، شماره مادر و شماره پدر. با زدن روی دکمه، شماره بدون نیاز به رفت‌وبرگشت با ربات مستقیماً در کلیپ‌بورد تلگرام کپی می‌شود (قابلیت `copy_text` تلگرام).
- **علامت‌گذاری خانه‌های زرد**: اگر پس‌زمینه‌ی سلول شماره در اکسل زرد باشد، روی دکمه‌ی مربوطه یک 🔵 (به‌جای آیکن معمولی) نمایش داده می‌شود.
  > تلگرام اجازه‌ی رنگی‌کردن واقعیِ پس‌زمینه‌ی دکمه‌های شیشه‌ای را نمی‌دهد (رنگ دکمه‌ها را کلاینت/تم کاربر کنترل می‌کند)، بنابراین این تمایز با ایموجی 🔵 شبیه‌سازی شده است.
- **دو دکمه‌ی وضعیت** («وضعیت پاسخگویی» و «حضور در جلسه‌ی مشاوره») که با هر بار لمس، به‌ترتیب بین چهار حالت تغییر می‌کنند:

  `⬜ خالی → ✅ تیک → ❌ ضربدر → 🔶 سایر → ⬜ خالی → ...`

  پیش‌فرض خانه‌های خالی اکسل هم «خالی» است.
- **جستجو**: در همان لیست پجینیشن‌شده، بر اساس نام یا بخشی از شماره تلفن (خودش/مادر/پدر/منزل).
- **خروجی اکسل آنی**: در هر مرحله می‌توان با یک دکمه یا دستور `/export`، فایل اکسلِ به‌روزشده (با همه‌ی تغییرات ثبت‌شده تا همان لحظه) را از ربات دریافت کرد. ساختار، استایل و رنگ‌های اصلی فایل حفظ می‌شود و فقط دو ستون وضعیت به‌روزرسانی می‌شوند.
- تشخیص ستون‌ها بر اساس **نام هدر** انجام می‌شود، نه موقعیت ثابت؛ چون ترتیب ستون‌ها می‌تواند بین شیت‌های مختلف یک فایل هم فرق کند.
- **تشخیص خودکار شیت‌ها**: نام و تعداد شیت‌ها هرچه باشد، مهم نیست. هر شیتی که ستون «ردیف» یا «نام و نام خانوادگی» داشته باشد، به‌عنوان یک شیت داده شناسایی و در منو نشان داده می‌شود؛ شیت‌های دیگر (مثل یک شیت خلاصه یا توضیحات) نادیده گرفته می‌شوند.
- **ثبت/تعویض فایل اکسل از داخل تلگرام**: نیازی به دسترسی به سرور نیست؛ کافی‌ست فایل `xlsx` را مستقیماً برای ربات ارسال کنید (توضیح کامل در بخش «ثبت یا تعویض فایل اکسل»).
- **آمار**: در متن هر صفحه، آمار وضعیت پاسخگویی و حضور برای کل همان شیت نشان داده می‌شود. دکمه/دستور `📊 آمار کلی` (`/stats`) هم آمار همه‌ی شیت‌ها را به‌همراه جمع کل نشان می‌دهد.

## پیش‌نیازها

- Python 3.10 یا بالاتر
- یک توکن ربات تلگرام از [@BotFather](https://t.me/BotFather)
- فایل اکسل با ستون‌های آشنا (نام‌ها می‌توانند کمی متفاوت باشند، تشخیص بر اساس هدر است نه موقعیت دقیق): ردیف، اپراتور، نام و نام خانوادگی، مقطع، رشته، نوع خدمات کلاس، تلفن منزل، تلفن دانش‌آموز، تلفن مادر، تلفن پدر، تاریخ تماس، وضعیت پاسخگویی، حضور در جلسه مشاوره، توضیحات. شیت‌ها می‌توانند هر تعداد و هر نامی داشته باشند.

## نصب (تک‌خطی)

این دستور فقط فایل‌های پروژه را دریافت می‌کند؛ اجراکردن ربات و ساخت محیط اجرایی (venv و…) با خودتان است:

```bash
git clone https://github.com/HellthonAriya/qzlexcel.git && cd qzlexcel && cp .env.example .env
```

برای نصب پیش‌نیازهای پایتون هم کافی‌ست این را (در محیط دلخواه خودتان) اجرا کنید:

```bash
pip install -r requirements.txt
```

## تنظیمات

فایل `.env` را باز کرده و مقادیر زیر را تنظیم کنید:

| متغیر | توضیح |
|---|---|
| `BOT_TOKEN` | توکن ربات از BotFather |
| `EXCEL_PATH` | مسیر فایل اکسل منبع (پیش‌فرض: `data/source.xlsx`) |
| `DB_PATH` | مسیر دیتابیس داخلی برای ذخیره‌ی وضعیت‌ها (خودکار ساخته می‌شود) |
| `PAGE_SIZE` | تعداد ردیف نمایش داده‌شده در هر صفحه (پیش‌فرض: ۵) |
| `ALLOWED_USER_IDS` | شناسه‌ی عددی کاربران مجاز تلگرام، جدا شده با کاما. خالی بگذارید یعنی همه مجازند (به‌دلیل حساس‌بودن اطلاعات تماس دانش‌آموزان، تنظیم این مقدار **قویاً توصیه می‌شود**) |

فایل اکسل را می‌توانید یا مستقیماً در مسیر `EXCEL_PATH` (مثلاً `data/source.xlsx`) قرار دهید، یا (ساده‌تر) پس از بالا آمدن ربات، آن را در چت تلگرام برای ربات ارسال کنید — بخش بعد را ببینید. پوشه‌ی `data/` در گیت نادیده گرفته می‌شود تا اطلاعات دانش‌آموزان هرگز در مخزن گیت‌هاب قرار نگیرد.

## ثبت یا تعویض فایل اکسل

اگر هنوز فایلی ثبت نشده باشد (مثلاً اولین اجرای ربات)، با زدن `/start` همین پیام را می‌بینید:

> ⚠️ هنوز هیچ فایل اکسلی ثبت نشده است. فایل اکسل موردنظر (فرمت xlsx) را همین‌جا برای ربات ارسال کنید تا ثبت شود.

برای **ثبت فایل اول** یا **تعویض فایل فعلی با یک فایل جدید**، کافی‌ست هر کاربر مجاز (اپراتور)، فایل `xlsx` را به‌صورت سند (Document) در همان چت برای ربات ارسال کند. ربات:

1. فایل را می‌خواند، خودش شیت‌های داده‌دار آن را (هرچه نامشان باشد) شناسایی می‌کند و تعداد ردیف‌های هر شیت را نشان می‌دهد.
2. برای تأیید نهایی، دو دکمه‌ی «✅ بله، جایگزین کن» و «❌ انصراف» نمایش می‌دهد.
3. با تأیید، فایل قبلی و **همه‌ی وضعیت‌های ثبت‌شده تا آن لحظه** جایگزین می‌شوند (چون ردیف‌بندی فایل جدید ممکن است با فایل قبلی یکی نباشد).

اگر ترجیح می‌دهید فایل را مستقیماً روی سرور جایگزین کنید (بدون ارسال در تلگرام)، فایل جدید را در مسیر `EXCEL_PATH` قرار دهید و دستور `/reload` را بزنید.

## دستورات ربات

| دستور | کاربرد |
|---|---|
| `/start` | نمایش منوی انتخاب شیت (بر اساس شیت‌هایی که در فایل فعلی شناسایی شده‌اند) |
| `/export` | دریافت فوریِ خروجی اکسل با آخرین تغییرات (همه‌ی شیت‌ها) |
| `/reload` | بارگذاری مجدد فایل اکسل از روی دیسک (برای وقتی که فایل منبع را دستی روی سرور جایگزین کرده‌اید) |
| `/reset_status` | پاک‌کردن همه‌ی وضعیت‌های ثبت‌شده در ربات |
| `/stats` | نمایش آمار پاسخگویی/حضور برای همه‌ی شیت‌ها به‌همراه جمع کل |

## ساختار پروژه

```
qzlexcel/
├── bot.py                # نقطه‌ی ورود ربات
├── app/
│   ├── config.py         # خواندن تنظیمات از .env
│   ├── excel_data.py     # خواندن/نگاشت هدر ستون‌ها/تجزیه‌ی وضعیت‌ها/خروجی اکسل
│   ├── storage.py        # ذخیره‌سازی وضعیت‌ها در SQLite
│   ├── data_store.py     # لایه‌ی حافظه‌ای که اکسل + وضعیت‌های ذخیره‌شده را ترکیب می‌کند
│   ├── keyboards.py       # ساخت متن و کیبورد شیشه‌ای هر صفحه
│   └── handlers.py       # هندلرهای تلگرام (start/export/reload/callback/جستجو)
├── data/                 # محل فایل اکسل و دیتابیس (در گیت نادیده گرفته می‌شود)
├── requirements.txt
└── .env.example
```

## عیب‌یابی

اگر هنگام اجرا این خطا را دیدید:

```
نسخه‌ی نصب‌شده‌ی python-telegram-bot قدیمی است و از دکمه‌ی کپی متن پشتیبانی نمی‌کند.
```

یعنی نسخه‌ی قدیمی‌تری از `python-telegram-bot` از قبل روی سیستم نصب بوده. این دستور را بزنید و دوباره ربات را اجرا کنید:

```bash
pip install --upgrade "python-telegram-bot>=21.3"
```

## نکات امنیتی

- فایل اکسل واقعی و دیتابیس وضعیت‌ها هرگز نباید به مخزن گیت‌هاب پوش شوند (شامل اطلاعات شخصی دانش‌آموزان هستند) — این مسیرها در `.gitignore` قرار دارند.
- حتماً `ALLOWED_USER_IDS` را با شناسه‌ی عددی اپراتورهای مجاز پر کنید تا افراد ناشناس نتوانند به لیست تماس‌ها دسترسی پیدا کنند.
