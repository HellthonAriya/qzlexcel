# qzlexcel — ربات تلگرام مدیریت پیگیری دانش‌آموزان

ربات تلگرامی برای نمایش، ویرایش و پیگیری وضعیت تماس با دانش‌آموزان از روی یک فایل اکسل سه‌شیتی (انسانی / تجربی / ریاضی)، با پجینیشن، جستجو، دکمه‌های شیشه‌ای کپی سریع شماره تلفن، و خروجی اکسل آنی از تغییرات.

## امکانات

- **پجینیشن**: نمایش ردیف‌ها صفحه‌به‌صفحه (تعداد قابل تنظیم در هر صفحه) با دکمه‌های اول / قبلی / بعدی / آخر.
- **متن هر صفحه**: دقیقاً مشخص می‌کند از چه شماره ردیفی تا چه شماره ردیفی (بر اساس ستون «ردیف» خود فایل) در حال نمایش است.
- **دکمه‌های شیشه‌ای کپی شماره**: برای شماره خودِ دانش‌آموز، شماره مادر و شماره پدر. با زدن روی دکمه، شماره بدون نیاز به رفت‌وبرگشت با ربات مستقیماً در کلیپ‌بورد تلگرام کپی می‌شود (قابلیت `copy_text` تلگرام).
- **علامت‌گذاری خانه‌های زرد**: اگر پس‌زمینه‌ی سلول شماره در اکسل زرد باشد، روی دکمه‌ی مربوطه یک 🔵 (به‌جای آیکن معمولی) نمایش داده می‌شود.
  > تلگرام اجازه‌ی رنگی‌کردن واقعیِ پس‌زمینه‌ی دکمه‌های شیشه‌ای را نمی‌دهد (رنگ دکمه‌ها را کلاینت/تم کاربر کنترل می‌کند)، بنابراین این تمایز با ایموجی 🔵 شبیه‌سازی شده است.
- **دو دکمه‌ی وضعیت** («وضعیت پاسخگویی» و «حضور در جلسه‌ی مشاوره») که با هر بار لمس، به‌ترتیب بین چهار حالت تغییر می‌کنند:

  `⬜ خالی → ✅ تیک → ❌ ضربدر → 🔶 سایر → ⬜ خالی → ...`

  پیش‌فرض خانه‌های خالی اکسل هم «خالی» است.
- **جستجو**: در همان لیست پجینیشن‌شده، بر اساس نام یا بخشی از شماره تلفن (خودش/مادر/پدر/منزل).
- **خروجی اکسل آنی**: در هر مرحله می‌توان با یک دکمه یا دستور `/export`، فایل اکسلِ به‌روزشده (با همه‌ی تغییرات ثبت‌شده تا همان لحظه) را از ربات دریافت کرد. ساختار، استایل و رنگ‌های اصلی فایل حفظ می‌شود و فقط دو ستون وضعیت به‌روزرسانی می‌شوند.
- تشخیص ستون‌ها بر اساس **نام هدر** انجام می‌شود، نه موقعیت ثابت؛ چون در فایل نمونه ترتیب ستون‌ها بین شیت «انسانی» و دو شیت دیگر متفاوت است.

## پیش‌نیازها

- Python 3.10 یا بالاتر
- یک توکن ربات تلگرام از [@BotFather](https://t.me/BotFather)
- فایل اکسل با همان فرمت (سه شیت با نام‌های «انسانی»، «تجربی»، «ریاضی» و ستون‌های: ردیف، اپراتور، نام و نام خانوادگی، مقطع، رشته، نوع خدمات کلاس، تلفن منزل، تلفن دانش‌آموز، تلفن مادر، تلفن پدر، تاریخ تماس، وضعیت پاسخگویی، حضور در جلسه مشاوره، توضیحات)

## نصب (تک‌خطی)

```bash
git clone https://github.com/HellthonAriya/qzlexcel.git && cd qzlexcel && python3 -m venv venv && source venv/bin/activate && pip install -r requirements.txt && cp .env.example .env
```

## تنظیمات

فایل `.env` را باز کرده و مقادیر زیر را تنظیم کنید:

| متغیر | توضیح |
|---|---|
| `BOT_TOKEN` | توکن ربات از BotFather |
| `EXCEL_PATH` | مسیر فایل اکسل منبع (پیش‌فرض: `data/source.xlsx`) |
| `DB_PATH` | مسیر دیتابیس داخلی برای ذخیره‌ی وضعیت‌ها (خودکار ساخته می‌شود) |
| `PAGE_SIZE` | تعداد ردیف نمایش داده‌شده در هر صفحه (پیش‌فرض: ۵) |
| `ALLOWED_USER_IDS` | شناسه‌ی عددی کاربران مجاز تلگرام، جدا شده با کاما. خالی بگذارید یعنی همه مجازند (به‌دلیل حساس‌بودن اطلاعات تماس دانش‌آموزان، تنظیم این مقدار **قویاً توصیه می‌شود**) |

فایل اکسل خود را در مسیر تعیین‌شده در `EXCEL_PATH` قرار دهید (مثلاً `data/source.xlsx`). این پوشه در گیت نادیده گرفته می‌شود تا اطلاعات دانش‌آموزان هرگز در مخزن گیت‌هاب قرار نگیرد.

## اجرا

```bash
source venv/bin/activate && python bot.py
```

## بروزرسانی (تک‌خطی)

```bash
cd qzlexcel && git pull && source venv/bin/activate && pip install -r requirements.txt --upgrade
```

## دستورات ربات

| دستور | کاربرد |
|---|---|
| `/start` | نمایش منوی انتخاب رشته (انسانی/تجربی/ریاضی) |
| `/export` | دریافت فوریِ خروجی اکسل با آخرین تغییرات (همه‌ی رشته‌ها) |
| `/reload` | بارگذاری مجدد فایل اکسل از روی دیسک (برای وقتی که فایل منبع را دستی جایگزین کرده‌اید) |
| `/reset_status` | پاک‌کردن همه‌ی وضعیت‌های ثبت‌شده در ربات (قبل از بارگذاری یک دیتاست کاملاً جدید از آن استفاده کنید) |

برای بارگذاری یک فایل اکسل کاملاً جدید (با ردیف‌بندی متفاوت)، ابتدا `/reset_status` و سپس فایل جدید را در `EXCEL_PATH` جایگزین کرده و `/reload` را بزنید؛ در غیر این صورت ممکن است وضعیت‌های قدیمی به‌اشتباه به ردیف‌های جدید نسبت داده شوند.

## ساختار پروژه

```
qzlexcel/
├── bot.py                # نقطه‌ی ورود ربات
├── app/
│   ├── config.py         # خواندن تنظیمات از .env
│   ├── excel_data.py     # خواندن/نگاشت هدر ستون‌ها/تجزیه‌ی وضعیت‌ها/خروجی اکسل
│   ├── storage.py        # ذخیره‌سازی وضعیت‌ها در SQLite
│   ├── data_store.py     # لایه‌ی حافظه‌ای که اکسل + وضعیت‌های ذخیره‌شده را ترکیب می‌کند
│   ├── keyboards.py       # ساخت متن و کیبورد شیشه‌ای هر صفحه
│   └── handlers.py       # هندلرهای تلگرام (start/export/reload/callback/جستجو)
├── data/                 # محل فایل اکسل و دیتابیس (در گیت نادیده گرفته می‌شود)
├── requirements.txt
└── .env.example
```

## نکات امنیتی

- فایل اکسل واقعی و دیتابیس وضعیت‌ها هرگز نباید به مخزن گیت‌هاب پوش شوند (شامل اطلاعات شخصی دانش‌آموزان هستند) — این مسیرها در `.gitignore` قرار دارند.
- حتماً `ALLOWED_USER_IDS` را با شناسه‌ی عددی اپراتورهای مجاز پر کنید تا افراد ناشناس نتوانند به لیست تماس‌ها دسترسی پیدا کنند.
